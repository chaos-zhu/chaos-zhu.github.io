  1. css in js

js沙箱机制： 在微前端环境中，子应用都可能用到window全局对象，因此在不同环境中可能对全局对象造成污染，怎么创造一个状态隔离的js运行环境

以下是qiankun源码中实现沙箱的两种方案(还原window的一段逻辑代码(快照))

  1. 快照隔离【通过new多个沙箱实例snapshotSandbox, 可以隔离多个不同应用的环境】

基本思路： 把window对象拷贝给一个新对象缓存下来(快照一份)，在退出的时候比对下原快照，然后把修改过的属性通过一个 modifyPropsMap 变量存起来，然后通过快照还原window

// js 快照沙箱
class SnapshotSandbox {
  constructor() {
    this.proxy = window
    this.modifyPropsMap = {} // 记录被修改的属性
    // this.active()
  }
  // 激活沙箱
  active() {
    this.windowSnapshot = {} // 用于存放激活时window的快照【每次激活需要清空一次, 否则可能遗留上次多余的属性】
    for (const key in window) {
      if (window.hasOwnProperty(key)) {
        this.windowSnapshot[key] = window[key] // 把window上的属性快照一份
      }
    }
    // 激活时如果发现原来有被更改过的属性，则重新赋值回来
    for (const key in this.modifyPropsMap) {
      window[key] = this.modifyPropsMap[key]
    }
  }

  // 沙箱失活(切换沙箱)
  inactive() {
    for (const key in window) {
      if (window.hasOwnProperty(key)) {
        if (window[key] !== this.windowSnapshot[key]) { // 如果window的某项属性不等于原来快照的属性
          this.modifyPropsMap[key] = window[key] // 则保存被更改过的属性
          window[key] = this.windowSnapshot[key] // 还原window激活时的属性
        }
      }
    }
  }
}

// 1号沙箱
const Sandbox1 = new SnapshotSandbox(); // 这个分号不能少，否则自调用函数(IIFE)报语法错误

Sandbox1.active() // 激活, 快照window属性
window.a = 1 // 新增属性
console.log(window.a) // 1

Sandbox1.inactive() // 失活, 还原window
console.log(window.a) // undefined

Sandbox1.active() // 再激活试试
console.log(window.a) // 1

Sandbox1.inactive() // 失活, 还原window，尝试使用2号沙箱


// =============
// 2号沙箱
const Sandbox2 = new SnapshotSandbox(); // 这个分号不能少，否则自调用函数(IIFE)报语法错误
Sandbox2.active()
window.a = 2
console.log(window.a) // 2
Sandbox2.inactive()
console.log(window.a) // undefined
Sandbox2.active()
console.log(window.a) // 2

Sandbox2.inactive() // 失活2号沙箱

Sandbox1.active() // 激活1号沙箱
console.log(window.a) // 1 查看1号沙箱的a属性
  1. ES6 Proxy隔离【使用Proxy代理window实例, 实现多应用隔离】
// js 代理沙箱
<script>
  class ProxySandbox{
    constructor() {
      const rawWindow = window
      const proxyWindow = {}
      this.proxy = new Proxy(proxyWindow, {
        get(target, key) { // target为第一个参数(代理对象proxyWindow)
          return target[key] || window[key]
        },
        set(target, key, value) {
          target[key] = value
          return true
        }
      })
    }
  }
  const ProxySandbox1 = new ProxySandbox()
  const ProxySandbox2 = new ProxySandbox()
  window.a = 1;
  // 沙箱1
  ((window) => {
    window.a = 2
  })(ProxySandbox1.proxy);

  // 沙箱2
  ((window) => {
    window.a = 3
  })(ProxySandbox1.proxy)

  // 代理沙箱，互不影响window
  console.log(window.a) // 1
</script>

# 使用 iframe 优缺点

# 优点

浏览器原生的硬隔离方案，完美的css隔离、js隔离

# 缺点

子应用切换路由，刷新页面iframe url丢失，后退前进功能无法使用(可以通过监听ifram url变化解决MutationObserver)

iframe嵌入的显示区大小不易控制，(例如：modal居中相对iframe大小，而不是浏览器窗口)

性能消耗大。每一个 iframe 都可以看成是一个小的浏览器窗口，应用每次进入相当于打开多个窗口。一般会比单页应用更慢。

    p1: 1,

  }

window.addEventListener('test', (params) => {
  console.log(params) // CustomEvnet对象
  console.log('params: ', params.detail) //  { p1: 1, p2: 2 }
})
// 1000ms后触发自定义事件【dispatchEvent为DOM对象上都存在的内置方法】
setTimeout(() => {
  window.dispatchEvent(test)
}, 1000)
</script>
